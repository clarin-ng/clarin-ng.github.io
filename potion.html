Encrypt

Secured Your Organized Content

Secure Your Files

Overview

Encrypt was the first capstone project of my Design apprenticeship at Bloc. It is a cloud-based storage application where user can perform standard cloud features like save and collaborate content. The main goal of Encrypt is to provide a secure and organized platform for users.

Problem

There are so many cloud storage products nowadays where people can store data and collaborate across the world online, however, when conducting research, I found out that there were two main areas that these platforms could improve.

  • Most users are worried about there data privacy. They are afraid others will access their content without permission.

  • People find it hard to organize their files with limited organizing tools.

Solution

Encrypt aims to create a secure and organized cloud-based platform that allows users to create a password for all types of content and also set an expiration date on a shared link to increase content’s privacy. Another great feature of Encrypt is users ability to organize content by tags, color ,or favorites.

Discovery & Strategy Phase

User Survey

A user survey was conducted with a total of 28 participants. The purpose was to determine which features are important to users and also to learn about the problems users encountered while using cloud storage.

Below are my key discoveries:

68.8%

68.8% of respondents use cloud storage services daily.

56.3%

56.3% of respondents said they use cloud storage services for personal matters. Others use it either for school or work.

Security

Improve the security of cloud storage will be the key to success. Respondents commented that they’re worried about the security of the applications. They do no trust platforms for mission criticals and tend not to save important files.

Organize

Respondents commented that they would like an application that could offer a better organization- the ability to sort manually.

Personas

After interviewing six users and combining the user survey data, two personas were created. Mark, as a personal user, often uses cloud storage to store and share pictures with his friend from a mobile. Amy tends to collaborate with her colleagues using a desktop. They both find it hard to organize their content and also worry about their data security.

Competitive Analysis

During the discovery phase, I identified three of the potential competitors- Google Drive, Dropbox, and Box. By analyzing each of their strength and weakness, this helped me realized their pros and cons and found out the areas a new market can improve and breakthrough.

  • Privacy settings that would allow users to secure their contents
  • Ability to organize with different grouping tools
  • A dashboard that is easy to use

Prototype & Testing

User Testing

User Testing I: Using InVision, I created a prototype with the black and white wireframes to begin testing 3 users remotely via Zoom to find out what issues I should fix.

User Testing II: I then created high-fidelity wireframes from the feedbacks that I recieved and tested the prototype with another 3 users.

Problems & Solutions

While the testings were very well, none of the users had problems with signing in and logging out; however, there were two consistent critiques:

  • The users felt confused about where to click to show more settings for the files. This led them to take several rounds of trial and error to perform the task of setting a password.
  • When going through the process of changing the dashboard to list view, the user took quite a while to figure this out.

Iteration #1

At first, the list view icon looked like the icon on the left; however, it looks like a dropdown menu to the users. Therefore, I redesigned the icon to the one on the right.

Iteration #2

I moved the location of the list view icon closer to “sort by” so that users could relate the icon as something that helped to organize their content.

Preference Testing

I conducted 2 preference tests on usability hub to find out if one design was favored the other.

Preference Testing #1: layout of the homepage

In this test, there was no significant difference; each option was 50%. Users chose option A because it required less scrolling time and easier to digest the information. Whereas people who chose option B because it’s more organized and less cluttered. I decided to move on with option B since “organized” was one of the Encrypt’s central theme.

Preference Testing #2:the color of the navbar

In this test, 60% of testers preferred option B. Their reasons were- it was more comfortable on the eyes, it looked softer and less weight than the black highlight (option A). I decided to move on with option B.

Conclusion-
what I learned

While working on this product, I found out that it was necessary to identify the main goals, because there were times that I wanted to add a lot of different features that were not necessary to solve the problem. Understand the goals kept me heading in the right direction to reach the MVP.


During user testing, it was interesting to see users usually had difficulty performing the first task and getting better with the next tasks because they were getting familiar with the product. I believe it would be better to randomize the sequence of tasks that the users were asked to perform to remove possible bias and uncover more potential issues.


For further development, I would want to validate the solution and find out if this could improve users frustration on data privacy and organization.
